# PART A — 마스터 컨텍스트

> A-1 ~ A-12를 **통째로 한 번** 붙여넣으세요. (기존 코드베이스가 있으면 "이 문서를 프로젝트의 기준 사양으로 삼아 줘"라고 덧붙입니다.)

## A-1. 역할

당신은 **시니어 프론트엔드 엔지니어 + 음악교육 UX 디자이너 + 플루트 초급 교재 설계자**를 겸합니다. 목표는 *"플루트 운지법과 계이름을 가장 직관적이고 사용자 친화적으로 배우는 모바일 웹앱"* 을 만드는 것입니다. **학습 콘텐츠의 음악적 정확성(A-7·A-9)은 절대 타협하지 않습니다. 추측으로 운지를 지어내지 말고, 제공된 데이터만 사용합니다.**

앱 이름: **플루트 핑거링 랩 (Flute Fingering Lab)**

## A-2. 제품 비전과 해결할 문제

한 문장 비전: **"계이름을 고르면, 누를 키가 악기 그림 위에 즉시 빛나고, 소리까지 들린다."**

이 앱이 반드시 해결하는 초보자의 문제:
1. 계이름 "도·레·미"를 보면 어떤 음(C·D·E)인지 모르겠다.
2. 음이름 "C4·D5"와 계이름 "도4·레5"가 연결되지 않는다.
3. 플루트 그림에서 **어떤 키를 눌러야 하는지** 직관적으로 안 보인다.
4. A♯/B♭처럼 이름은 둘인데 소리는 같은 음, 대체 운지가 헷갈린다.
5. 저·중·고음역에서 운지가 비슷해 보여도 호흡·소리내기가 다르다.
6. 내가 분 음이 맞는지 **즉시 피드백**을 받고 싶다.

5대 UX 원칙:
1. **보이는 것이 먼저** — 글보다 큰 플루트 그림과 색으로 설명.
2. **즉각 피드백** — 음 선택 0.1초 내 운지 하이라이트 + 실음 재생.
3. **점진적 공개** — 초보 화면은 단순하게, 트릴키·대체운지는 펼치기로.
4. **한 손 모바일** — 보면대에 폰을 세워 두고 한 손으로. 탭 영역 ≥ 44px, 하단 탭바.
5. **연결해서 가르치기** — 오선보 음표 ↔ 음이름 ↔ 계이름 ↔ 운지 ↔ 소리를 한 화면에서 연결.

## A-3. 타깃 사용자 · 학습 목표 · 추천 학습 경로

- **사용자:** 플루트를 처음 배우는 청소년·성인, 방과후/동아리, 독학자, 모바일로 빠르게 운지를 찾는 사람.
- **학습 목표(앱을 다 쓰면 할 수 있는 것):** ① C4(가온다)~C7에서 임의 음의 운지를 안다 ② 음이름↔계이름 관계를 이해한다 ③ 오선보의 음을 보고 이름·운지를 떠올린다 ④ 간단한 동요를 운지 가이드로 끝까지 분다.
- **추천 학습 경로(홈에 "오늘의 5음"으로 노출):**
  - 1일차: 도4–솔4 / 2일차: 라4–도5 / 3일차: 반음과 시♭ / 4일차: 중음역 D5–G5 / 5일차: 간단한 멜로디(작은별).

## A-4. 기술 스택

- **프레임워크:** React 18 + TypeScript + **Vite** (단일 SPA로 충분). PWA 지원.
- **스타일:** **Tailwind CSS** + CSS 변수 디자인 토큰(A-6).
- **애니메이션:** **Framer Motion** (키 눌림, 화면 전환, 정답 피드백). `prefers-reduced-motion` 존중.
- **상태/저장:** **Zustand** (+ persist), 진도·설정은 `localStorage`. 백엔드 없음.
- **데이터:** `src/data/notes.ts` 정적 객체(A-9)가 **유일한 정답 출처(Source of Truth)**.
- **오디오:** 1순위 **`smplr`** 의 Soundfont 플루트(실음에 가까움), 폴백으로 **Web Audio API 신디사이저**(A-9·Phase 4 사양). 외부 mp3/wav 의존 금지.
- **오선보:** **VexFlow** (높은음자리표·음표·**덧줄 자동 생성**).
- **아이콘:** `lucide-react`. **폰트:** 한글 가독성을 위해 **Pretendard**.

> 오디오(smplr)와 오선보(VexFlow)는 직접 구현하지 말고 라이브러리를 사용하세요. (단, 오디오 폴백 합성기는 Web Audio API로 직접 구현.)

## A-5. 정보구조(IA)와 화면

**하단 고정 탭 5개**(모바일 네이티브 느낌):

1. **홈** — 오늘의 5음, 빠른 검색, 추천 경로 카드.
2. **운지 탐색(Explore)** — 핵심. 음 선택 → 플루트 운지 + 음이름/계이름/오선보/주파수/연주팁/소리. **키를 직접 짚어 음을 역추적**하는 모드 포함(A-10).
3. **배우기(Learn)** — 1카드=1개념: ① 플루트 잡는 법·키 이름 ② 음이름 vs 계이름 ③ 첫 5음 ④ 옥타브와 과학적 음높이.
4. **연습(Practice)** — 스케일·동요 따라불기(음표가 진행되며 운지 자동 전환, BPM·step 모드).
5. **퀴즈(Quiz)** — 게임형 3유형 + 스트릭·타이머.

상단 공통 컨트롤: **음역 토글(저/중/고)**, **표기 토글(음이름/계이름/둘 다)**. 설정은 홈 또는 우상단 아이콘으로 진입.

## A-6. 디자인 시스템

크림 종이 + 차분한 네이비 잉크 + 골드(brass) 포인트의 우아한 톤. CSS 변수로 정의해 Tailwind에서 사용.

```css
:root{
  --paper:#f7f6f3;      /* 페이지 배경(크림) */
  --card:#ffffff;       /* 카드 */
  --ink:#2a3342;        /* 주 텍스트 · 눌린 키(네이비) */
  --ink-2:#6b7280;      /* 보조 텍스트 */
  --brass:#9a6b3f;      /* 강조(제목·옥타브·포인트, 골드) */
  --tube:#ecedef;       /* 플루트 관(실버) */
  --tube-line:#c7ccd1;  /* 관 테두리 · 열린 키 보조선 */
  --hair:#e3e1db;       /* 카드 경계선 */
  --ok:#2f8f6b;         /* 정답 */
  --warn:#c2543d;       /* 오답 */
  --hl:#f3c969;         /* 진행 중 음표 하이라이트(골드) */
  --radius:16px;
}
```

- **타이포:** Pretendard. 음 이름은 28~34px/800, 보조 라벨 12~13px/`--ink-2`. **본문 최소 16px**(iOS 입력 시 자동 확대 방지).
- **키 표현 규칙(가장 중요·전 화면 일관):** **누름 = 꽉 찬 원(`--ink`)**, **뗌 = 흰 원 + 테두리.** 색에만 의존하지 말고 **형태 차이(채움/약간 큰 반지름)와 텍스트 라벨**도 함께 제공.
- **모션:** 키 눌림 `scale .92→1` + 색 채움(150ms). 정답 초록 펄스, 오답 좌우 흔들림. `prefers-reduced-motion`이면 모두 끔.

## A-6.5. 모바일 UI/UX 상세 (★ 이 앱의 최우선 설계 기준)

> 사용 맥락: **악보대에 폰/태블릿을 세워 두고, 한 손으로, 연습 중에** 쓴다. 아래를 빠짐없이 반영.

**레이아웃 · 내비게이션**
- 단일 컬럼 + 세로 스크롤. **하단 고정 탭바**(5개), `padding-bottom: env(safe-area-inset-bottom)`로 홈 인디케이터 회피.
- 상세(운지) 화면 하단에 **스티키 액션 바**(재생 · 한 단계씩 · 연습에 추가) — 엄지로 닿는 화면 하단 1/3에 배치.
- 노치/상태바: `env(safe-area-inset-top)` 고려. 가로 스크롤 금지(음 선택 칩 스트립만 의도적 가로 스크롤 + 관성).

**터치 타깃 · 히트박스**
- 모든 탭 요소 **최소 44×44px**(권장 48dp), 간격 ≥ 8px.
- **SVG 키 히트박스 확대:** 보이는 키보다 넓은 투명 패딩 레이어를 키 위에 깔아 **약간 빗나간 탭도 인식**(`pointer-events` 투명 원). 모바일 오타 방지의 핵심.

**플루트 다이어그램 표시**
- 기본 가로형, `width:100%` + `aspect-ratio`로 레이아웃 시프트 방지(`viewBox` 비율 유지).
- **세로 회전 보기** 토글 제공(작은 폰에서 키가 더 크게). 핀치 줌은 선택.
- 가로 모드(landscape)에서는 `flex-col lg:flex-row`로 다이어그램을 키우고 정보 패널을 옆에 둔다. 화면 회전 시 **잘리지 않게**.

**입력 · 제스처**
- **호버 의존 금지** — 모든 정보는 탭 또는 상시 표시. 호버는 데스크톱 보강용으로만.
- 상세 화면에서 **좌/우 스와이프 = 이전/다음 음**, 연습에서 스와이프 = 다음 음(옵션).
- 햅틱: 정답/오답·키 탭에 `navigator.vibrate?.(10)`(지원 기기에서만, 은은하게).

**오디오(모바일 정책 방어) — 매우 중요**
- iOS/Android는 사용자 제스처 없이 소리를 막는다. **첫 탭 또는 "악기 깨우기(소리 켜기)" 버튼**에서 `AudioContext.resume()` 실행 + smplr 샘플 로드. 로딩 중 "소리 준비 중" 인디케이터 → 준비되면 자동 재생 가능.
- 무음 모드/저전력에서의 한계는 토스트로 안내. 샘플은 **첫 상호작용 이후 지연 로드**(초기 로딩 가볍게).

**성능 · 안정성**
- `<FluteDiagram>`은 `React.memo`로 불필요한 리렌더 차단. SVG는 가볍게.
- `localStorage`가 비거나 깨져 있어도 앱이 정상 구동(방어적 파싱).

**설치 · 접근성**
- **PWA**: 설치 가능(standalone), `theme-color`, 오프라인 셸 캐시, "홈 화면에 추가" 힌트.
- 키보드 탐색·포커스 링, 모든 인터랙티브 요소 `aria-label`, 명도 대비 확인, 색 외 형태 단서.

## A-7. 음악적 정확성 규칙 (절대 준수)

기준 악기: **표준 베임(Boehm) 시스템 · 폐쇄 G♯(closed G#) C 플루트.**

1. **음역:** 표준 C풋 기준 **C4 ~ C7**. **B3** 는 B풋 조인트가 있어야 가능 → "B풋 전용"으로 표시(설정에서 C풋이면 비활성).
2. **비이조 악기:** **기보음 = 실음**, **C4 = 가온다(middle C)**. 과학적 음높이 표기(C4, D5…).
3. **E♭(D♯) 키는 기본 벤트:** 대부분의 음에서 오른손 새끼가 **E♭ 키를 누른 채** 분다. 단 **D4·D5는 E♭ 키를 뗀다**(D의 특징). 최저음 **B3·C4·C♯4**, 일부 최고음(**B♭6·B6·C7**)도 E♭를 누르지 않는다. → A-9 데이터의 `keys`에 명시된 대로만 칠한다.
4. **B♭(A♯)는 운지 두 가지:** ① **엄지 B♭ 레버(Briccialdi)**(권장 기본) ② **오른손 검지(1+1)**. 둘 다 제공, 기본은 ①.
5. **C7도 두 가지:** ① **C풋(모든 악기 가능, 기본)** ② **B풋 + 기즈모(gizmo)**. 둘 다 제공.
6. **F♯6 주의:** 엄지는 반드시 **B 내추럴 레버**(B♭ 레버 아님). 각주 안내.
7. **추측 금지:** A-9에 없는 운지를 만들지 말 것. 의심되면 내부 주석 `needsReview`만 남기고 사용자에겐 확정 표기. 출처: **The Woodwind Fingering Guide**. (알티시모 4옥타브는 악기·연주자별 편차가 커 **기본 범위에서 제외**, 확장은 C-7 참고.)

## A-8. 데이터 모델 (TypeScript) — `src/data/types.ts`

```ts
// 16개 키 식별자 (플루트 그림의 각 키와 1:1)
export type KeyId =
  | "thumbB" | "thumbBb"            // 왼손 엄지: B 내추럴 / B♭ 레버
  | "lh1" | "lh2" | "lh3" | "gs"    // 왼손 검지·중지·약지 / G♯(새끼)
  | "rh1" | "rh2" | "rh3"           // 오른손 검지·중지·약지
  | "dTrill" | "dSharpTrill"        // D 트릴 / D♯ 트릴
  | "eb"                            // E♭(D♯) 키 (기본 벤트, 오른손 새끼)
  | "csFoot" | "cFoot" | "bFoot"    // 풋 조인트: 저 C♯ / 저 C / 저 B
  | "gizmo";                        // 기즈모(고음 C 보조)

export type Register = 1 | 2 | 3;   // 저 / 중 / 고 (운지 패턴 기준)

export interface Fingering {
  keys: KeyId[];            // 눌러야 하는 키(정답). 이 배열이 SVG·토큰·단계의 단일 출처
  recommended?: boolean;    // 대체 운지 중 추천
  requiresBFoot?: boolean;  // B풋 악기 필요
  requiresGizmo?: boolean;  // 기즈모 필요
  altLabel?: string;        // 대체 운지 이름(예: "오른손 검지(1+1)")
}

export interface FluteNote {
  id: string;               // "C4" (과학적 음높이, C4 = 가온다)
  letter: "C"|"D"|"E"|"F"|"G"|"A"|"B";
  accidental?: "sharp"|"flat";
  octave: number;           // 3~7 (B3는 3)
  midi: number;             // 60 = C4
  enharmonic?: string;      // 딴이름한소리(예: "D♭4")
  register: Register;
  fingerings: Fingering[];  // [0] = 기본 운지
  note?: string;            // 표시용 각주
}

// 주파수는 MIDI에서 계산 (하드코딩 금지)
export const freqOf = (midi: number) => 440 * 2 ** ((midi - 69) / 12);
```

```ts
export function keysToToken(keys: KeyId[]) {
  const has = (k: KeyId) => keys.includes(k);
  const thumb = has("thumbBb") ? "B♭T" : has("thumbB") ? "T" : "";
  const left  = (["lh1","lh2","lh3"] as KeyId[]).map(k => has(k)?TOK[k]:"-").join("") + (has("gs")?"G#":"");
  const right = (["rh1","rh2","rh3"] as KeyId[]).map(k => has(k)?TOK[k]:"-").join("");
  const extra = (["dTrill","dSharpTrill","eb","csFoot","cFoot","bFoot","gizmo"] as KeyId[])
    .filter(has).map(k=>TOK[k]).join("");
  return `${thumb} ${left} | ${right}${extra}`.replace(/\s+/g," ").trim();
}

// 4) 고정도법 계이름
const SOL: Record<string,string> = {C:"도",D:"레",E:"미",F:"파",G:"솔",A:"라",B:"시"};
export function solfegeFixed(letter: string, acc?: "sharp"|"flat") {
  const b = SOL[letter];
  return !acc ? b : acc==="sharp" ? `올림${b}(${b}♯)` : `내림${b}(${b}♭)`;
}

// 5) 검색 별칭 생성 (도4 / 도 / c4 / C4 / do4 / 시b / 시♭ …) — A-12 규칙
const ROMA: Record<string,string> = {도:"do",레:"re",미:"mi",파:"fa",솔:"sol",라:"la",시:"si"};
export function buildAliases(n: {id:string; letter:string; accidental?:string; octave:number; enharmonic?:string}) {
  const out = new Set<string>();
  const base = SOL[n.letter];
  const accK = n.accidental==="sharp" ? "♯" : n.accidental==="flat" ? "♭" : "";
  const accA = n.accidental==="sharp" ? "#" : n.accidental==="flat" ? "b" : "";
  out.add(n.id); out.add(n.id.toLowerCase());
  out.add(`${n.letter}${accA}${n.octave}`); out.add(`${n.letter}${accA}${n.octave}`.toLowerCase());
  out.add(`${base}${accK}${n.octave}`); out.add(`${base}${accK}`); out.add(`${base}${n.octave}`);
  out.add(`${base}${accA}${n.octave}`);                 // 시b4
  out.add(`${ROMA[base]}${accA}${n.octave}`);           // sib4
  if (n.enharmonic) out.add(n.enharmonic);
  return [...out];
}
```

## A-12. 계이름·음이름 매핑 & 검색 정규화

- 기본 **고정도법(fixed-do)**: C=도 … B=시. 임시표는 `올림도(도♯)/내림미(미♭)` 식 병기(C-1 표).
- **검색 정규화(한글 친화):** 입력을 음 토큰으로 변환 후 `aliases`와 매칭.
  - `도→C, 레→D, 미→E, 파→F, 솔→G, 라→A, 시→B`

- 표기 토글: 초보자 모드는 `도4`를 크게/`C4` 작게, 고급자 모드는 동일 비중.

## C-2. 검색 별칭 정규화 규칙

```text
계이름→음이름:  도→C 레→D 미→E 파→F 솔→G 라→A 시→B
임시표:        샵/#/sharp→#     b/♭/플랫/flat→b
조합 예:        도#→C#   레b→Db   시b→Bb   라#→A#
```

## C-4. 음역별 연주 팁 (요약 — 전체는 A-9-2 `octaveTips`)

- **저음역(1):** 바람을 낮고 넓게, 입술을 너무 조이지 않기. 소리가 위로 튀면 바람 속도 점검.
- **중음역(2):** 바람 중심을 조금 더 모으고, 음 시작을 '투/두'로 부드럽게.
- **고음역(3):** 바람 속도↑·목/어깨 힘 빼기, 작은 구멍·빠른 바람. 짧게 시도하고 교사 확인 권장.
